### Reflection on Design Principles

In this lab, I applied basic software design principles while sketching the UML diagrams for a simple online bookstore system.

**Abstraction** was used in the Use Case Diagram by showing only the main actions a customer can perform (Browse Books, Add to Cart, Checkout) without showing any internal implementation details.

**Modularity** is visible in the Class Diagram, where the system is broken into separate classes: Book, Customer, ShoppingCart, and Order. Each class handles its own responsibility.

**Cohesion** is maintained because each class has a single, clear purpose. For example, the ShoppingCart class only deals with adding books and calculating the total.

**Coupling** is kept low by using simple relationships between classes instead of making them heavily dependent on each other.

Overall, these diagrams help demonstrate how a clean and organized design makes the system easier to understand and build.
